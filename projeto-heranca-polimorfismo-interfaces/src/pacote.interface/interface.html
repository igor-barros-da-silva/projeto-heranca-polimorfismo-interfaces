Introdução a Interfaces

  • Uma interface pode definir uma série de métodos, mas nunca conter implementação deles.
  • Ela só expõe o que o objeto deve fazer, e não como ele faz, nem o que ele tem.
  • Como ele faz vai ser definido em uma implementação dessa interface.
Lemos a interface da seguinte maneira:

*"quem desejar entrar no sistema precisa obrigatoriamente seguir uma regra dentro so sistema."

Ela é um contrato onde quem assina se responsabiliza a implementar esses métodos.

Problema:
Queremos que apenas o secretário do nosso sistema pode acessar o sistema.
Solução:
O secretário deve implementar uma interface que mostra, obriga e realiza essa liberação.

O método
public interface permitirAcesso() {return boolean autenticar()}
Abstração e Herança

Usando o nosso exemplo de uma escola se escrevermos um método abstrato ou então escrever o método de Permitir acesso na classe Pessoa e usarmos a herança, todos os tipos de pessoas poderaõ acessar e isso não queremos permitir.
É um erro grave pensar que a herança e abstração irá resolver esse caso em 100%.
Resolve mas causam vários outros problemas.
Interface
Usando a interface conseguimos fazer um contrato com o tipo Secretário e obrigar apenas a um tipo de pessoa a acessar nosso sistema e com isso todos os outros tipos de pessoa não sabem absulutamente nada sobre a permissão de acesso.
Caso mais frente seja necessário permitir acesso ao Diretor então ele implementará a interface também.
Só fazemos o contrato e expomos essa permissão para quem for necessário.
