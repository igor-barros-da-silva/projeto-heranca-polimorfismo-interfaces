O que é uma exceção?

Quando criamos sistemas podem ocorrer erros imprevistos durante sua execução, esses erros são conhecidos como exceções e podem ser erros de lógica, conversões, acessos e etc. É muito difícil e quse impossível pra o programador prever todos os erros que podem acontecer nesse ponto o tratamento de exceções são obrigatórios para o sistema continuar funcionando. O tratamento da exceção: Fazendo e prevendo que algo pode acontecer os sistemas se recuperam e continuam a funcionar para o usuário. O NÃO tratamento da exceção: Isso acontece muito em sistemas e ocasiona a parada do sistema e com isso em um banco por exemplo irá ocasionar filas de clientes. Em uma empresa pode parar totalmente a produção da fábrica.

Fatores internos e externos

A origem de um erro pode ser interna, problema no próprio código e esse erro pode ser corrigido rapidamente pela equipe. Outra origem pode ser externa o que muitas vezes a equipe de desenvolvimento não tem domínio e controle do que acontecer restando apenas informar aos usuários o motivo. Alguns motivos externos: Tentar abrir um arquivo que não existe. Tentar fazer consulta a um banco de dados remoto que não está disponível. Tentar escrever algo em um arquivo sobre o qual não se tem permissão de escrita. Tentar conectar em servidor inexistente ou fora de serviço. Alguns motivos internos: Tentar manipular um objeto que está com valor nulo. Dividir um número por zero. Tentar manipular um tipo de dado com se fosse outro. Tentar um utilizar um método ou classe não existente.

Arquitetura Java da Exceções

Compilação: Erro antes da fase de execução. Impede de iniciar o sistema. Runtime: Erro durante a execução. Impede uma rotina do sistema.

Estrutura do tratamento de Exceções

Dentro do TRY fica o sódigo que pode ocorrer uma exceção e CATCH é o ponto de caída do código após a interrrupção. A classe EXCEPTION tem todas as informações do erro. Toda exceção interrompe um bloco em um fluxo de processo. try { // Códifo de regra de negócio - processos do sistema. // ex. realização de uma venda. } catch (Exception e) { e.printStackTrace(); // IMPRIME A PILHA DE ERRO NA CONSOLE }

Erros mais comum para iniciantes

1 - NullPointerException: Erro número 1 do Java quer dizer existe uma variável ou referência nula. 2 - NumberFormatException: Tentando converter uma string em um número mas a string estava vazia. 3 - FileNotFoundException: Tentando carregar um arquivo mas não foi encontrado no caminho informado. 4 - ClassCastException: Tentando converter um objeto B para A sendo que não são do mesmo tipo ou sem relação. 5 - classNoFoundException: Tentando carregar uma classe sendo que não foi encontrada. Existem finitas exceções.
